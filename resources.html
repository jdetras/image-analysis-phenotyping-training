Referenced software

Every tool retains its own license, summarised on the Setup page. A few warrant attention for organisations:

  • Ultralytics YOLO is AGPL-3.0. That is fine for training, research, and teaching, but using it inside a closed-source product can trigger copyleft obligations — Ultralytics sells a commercial license for that case. Evaluate before embedding it in proprietary pipelines.
  • Fiji, ilastik, CloudCompare, QGIS are GPL-family — free to use and share; modifications you distribute must stay open.
  • SAM, Open3D, PlantCV, scikit-image, scikit-learn, PyTorch, OpenCV are permissive (Apache-2.0 / BSD / MIT) — minimal restrictions.
Not legal advice. License summaries here are provided in good faith for planning. Confirm current terms on each project's site before redistributing software or building commercial products.

Hosting this site

This is a static site — no build step, no server-side code. To publish it on GitHub Pages:

  1. Create a GitHub repository and push this folder to it (see the repository README.md for the exact commands).
  2. In the repo, go to Settings → Pages.
  3. Set Source to Deploy from a branch, branch main, folder / (root).
  4. Save. Your site goes live at https://<user>.github.io/<repo>/ within a minute or two.

The .nojekyll file tells GitHub Pages to serve the HTML as-is without running Jekyll, so there is nothing to break.
